Validation 2: BC-Aus vs CER reference (no-offset variant, 0°F)

Compares the calculator's Burch-Christensen mains water model with CER (Clean Energy Regulator) monthly reference values.
Ready. Start the local backend at http://localhost:8000/tmy and then run the validation.
CER = fixed monthly reference.
Model = TMY-based calculation.
Click a zone to view its chart.

How It Works

1

CER Data

Monthly cold water values come from the local CER TRNSYS deck files in validation/.
2

TMY Input

The page fetches TMY weather from the local /tmy endpoint for each zone.
3

Model

It runs the same Burch-Christensen calculation as index.html and compares month by month.

Zone Summary

Click a zone name for its chart.

Zone Detail

Solid circles show the CER reference temperatures. Dashed triangles show the model prediction from the TMY-driven Burch-Christensen calculation.

Source

CER reference values were extracted from the local TRNSYS deck files in validation/: zone1_NW_Domestic.inc, zone2_NW_Domestic.inc, zone3_NW_Domestic.inc, zone4_NW_Domestic.inc, and ZONEHP5_Au_Domestic.inc.

The files include the note Revised 20/07/15. These values are certification schedules, not one measured calendar year.

Model weather comes from PVGIS TMY via the local backend. A TMY is a representative year, not one specific year.

Official CER source: https://cer.gov.au/schemes/renewable-energy-target/renewable-energy-target-participants-and-industry/manufacturers-and-importers

PROBLEM
The model matches the seasonal shape well, but it tends to over-predict winter mains temperatures, especially in cooler inland zones. A likely reason is that the original NREL correlation was developed for US conditions, while Australian pipe depth, soil, and network conditions may differ.
