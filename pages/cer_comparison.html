5 AS/NZS 4234 Climate Zones Weather input: CER TMY + PVGIS (our API that pulls weather)
Info

AS/NZS 4234: The Australian/New Zealand standard for solar hot water system certification. Defines 5 climate zones and reference mains temperatures used by the Clean Energy Regulator (CER) for STC calculations.

CER TMY (solid lines): Formula-only check — uses CER's own weather files, same data that produced the green reference line. Isolates formula bias.

PVGIS (dashed teal line): Live-calculator check — what the main calculator would actually predict for a real user at this location. Fetch per zone using the button.

BC-Aus: Australian refit of Burch-Christensen. Run python3 fit_bc_aus.py to regenerate bc_aus_constants.js.

Regional BC-Aus: Separate five-constant fit for each AS/NZS 4234 climate zone. This is the stronger regional version than changing only the offset. Run python3 fit_bc_aus_by_zone.py to regenerate bc_aus_zone_constants.js.

Zone correction: Exact monthly correction vectors still exist in bc_zone_corrections.js for the perfect-fit benchmark, but the main chart below now focuses on the pure national and regional BC formula fits.

PVGIS weather input

BC-Aus constants — shared national fit replacing the US Burch-Christensen defaults, fitted to Australian CER data via fit_bc_aus.py

A Python script (fit_bc_aus.py) adjusts all five parameters simultaneously — offsetF, ratio_c0, ratio_c1, lag_c0, lag_c1 — to find the combination that best matches the Australian CER reference temperatures across all 60 data points (5 zones × 12 months) from the CER DomDecks V30h climate files.

Regional per-zone BC-Aus constants are loaded separately from bc_aus_zone_constants.js and are not changed by these manual national inputs.
